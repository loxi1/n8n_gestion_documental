Cómo debe quedar el flujo
No en una sola línea secuencial simple. Debe haber dos Merge reales.
Hazlo así:
Plain text
Code original
   ├─→ Write File
   └─→ Guardar_Correo
          ↓
     Code_correo_id
Luego:
Plain text
Code original ─┐
               ├─→ Merge_1 (combina metadata original + correo_id)
Code_correo_id ┘
Después:
Plain text
Merge_1 → Guardar_Documentos → Code_documento_id
Y luego:
Plain text
Merge_1 ───────────────┐
                       ├─→ Merge_2 (combina metadata + correo_id + documento_id)
Code_documento_id ─────┘
Finalmente:
Plain text
Merge_2 → Guardar_Archivos
Qué hacen los dos Code pequeños
Después de Guardar_Correo:
JavaScript
return items.map(item => {
  item.json.correo_id = item.json.id;
  delete item.json.id;
  return item;
});
Después de Guardar_Documentos:
JavaScript
return items.map(item => {
  item.json.documento_id = item.json.id;
  delete item.json.id;
  return item;
});
Cómo configurar los Merge
Usa el nodo Merge real de n8n, no un Code llamado “merge”.
Configúralo para combinar por posición si cada PDF genera exactamente 1 fila en cada rama.
Como tu flujo actual trabaja 1 PDF → 1 correo/documento, “by position” es lo más simple.
