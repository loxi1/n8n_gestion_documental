Perfecto, ya encontramos el problema exacto:
Qué pasa
Tu ocrmypdf está instalado en:
Plain text
/root/venvs/ocrpdf/bin/ocrmypdf
pero cuando llamas wsl ... desde PowerShell o desde Python, WSL entra con tu usuario por defecto, no como root.
Entonces intenta ejecutar un archivo dentro de /root/... y por eso da:
Plain text
Permission denied
La mejor solución
Mueve el entorno OCR al usuario normal loxi1
No lo dejes en /root.
Eso te va a evitar problemas ahora y después.
Haz esto dentro de WSL
Entra a WSL como tu usuario normal loxi1, no como root.
Primero verifica:
Bash
whoami
echo $HOME
Debería salir algo como:
Plain text
loxi1
/home/loxi1
Luego crea el nuevo venv en tu home
Bash
mkdir -p ~/venvs
python3 -m venv ~/venvs/ocrpdf
source ~/venvs/ocrpdf/bin/activate
pip install --upgrade pip
pip install ocrmypdf pypdf
Verifica
Bash
which ocrmypdf
ocrmypdf --version
Debería quedar algo como:
Plain text
/home/loxi1/venvs/ocrpdf/bin/ocrmypdf
Luego prueba desde PowerShell
PowerShell
wsl bash -lc "/home/loxi1/venvs/ocrpdf/bin/ocrmypdf --version"
Si responde bien, ya quedó solucionado.
Qué cambiar en tu procesar_pendientes.py

Python

cámbialo por:

f'/home/loxi1/venvs/ocrpdf/bin/ocrmypdf -l spa --force-ocr "{input_wsl}" "{output_wsl}"'
