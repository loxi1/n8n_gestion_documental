Matriz final de decisión
1. Si el correo tiene una factura válida
Condiciones mínimas:
tipo factura
RUC extraído o razonablemente identificable
fecha de emisión identificable
Entonces:
Acción
esa factura será el documento principal
se genera grupo_codigo tipo 04-0001
se toma el mes desde fecha_emision
se clasifican y renombran los demás adjuntos del correo
Destino
carpeta: pendientes_clasificados/YYYY/MM/
estado correo: procesado
estado documento: clasificado
estado archivo: renombrado
2. Si el correo tiene factura legible y factura escaneada del mismo comprobante
Condiciones:
ambas pertenecen al mismo correo
mismo RUC emisor
una tiene serie+número y la otra no
Entonces:
Acción
la factura legible = factura
la escaneada = adjunto_factura
Destino
ambas a pendientes_clasificados/YYYY/MM/
grupo común
la legible como principal
3. Si el correo tiene factura y además OC / guía / requerimiento / cotización
Entonces:
Acción
factura = principal
OC = orden_compra
guía = guia_remision
requerimiento = requerimiento_compra
cotización = cotizacion
otros PDFs = adjunto_documento
Destino
todo a pendientes_clasificados/YYYY/MM/
mismo grupo_codigo
4. Si el correo no tiene factura, pero sí documentos útiles
Ejemplos:
solo guía
solo OC
solo certificado
solo cotización
solo requerimiento
Entonces:
Acción
no se genera grupo automático
sí se guarda el tipo documental detectado
sí se guarda RUC / serie / número / fecha si existen
queda pendiente de asociación manual
Destino
carpeta: pendientes_revision/YYYY/MM/
estado correo: sin_factura_principal
estado documento: pendiente_asociacion
estado archivo: observado
5. Si hay factura pero no tiene fecha reconocible
Entonces:
Acción
no clasificar automáticamente
guardar todo lo extraído
mandar a revisión
Destino
carpeta: pendientes_revision/YYYY/MM/
estado correo: factura_sin_fecha
estado documento: revision_manual
estado archivo: observado
6. Si el documento no se entiende
Ejemplos:
PDF vacío
OCR fallido
contenido irreconocible
archivo extraño
Entonces:
Acción
marcar como no identificado
guardar nombre original y cualquier metadato mínimo
Destino
carpeta: no_identificados/YYYY/MM/
estado documento: no_identificado

Reglas de carpeta
inbox/
Solo lo recién descargado por n8n.
pendientes_clasificados/
Solo lo ya resuelto correctamente.
pendientes_revision/
Todo lo válido pero que necesita revisión humana.
no_identificados/
Lo que no se pudo entender.
ocr_tmp/
Solo temporales OCR.
Regla de oro del sistema
Si hay factura válida
Automatiza.
Si no hay factura válida
Revisión.
Si no se entiende
No identificado.
Regla de mes
Siempre usa:
fecha_emision
Nunca:
fecha del correo
fecha de recepción
fecha del archivo
Regla de cliente destino
Se resuelve por:
cliente detectado en la factura
alias de cliente destino
si no, queda para revisión
Regla para red compartida
Todavía no se aplica.
Primero debe quedar estable:
n8n descarga bien
BD guarda bien
procesar_pendientes.py clasifica bien
carpetas locales quedan correctas
Después se hace la copia/movimiento a la red.
Checklist para dar por estable la lógica local
Debe cumplirse esto en varias pruebas seguidas:
factura FE65 sale como factura
guía sola sale como guia_remision y va a revisión
certificado sale como certificado_calidad y va a revisión
grupos 04-000x no se reprocesan
adjuntos escaneados salen como adjunto_factura
clasificados no vuelven a pendientes_revision

Tu lógica congelada debe ser esta:
