La OC ya se detecta, pero estás ejecutando _extract_oc_fields() también para facturas/guías, por eso ves SERIE: OC en documentos que no son OC. Eso no debe usarse para cambiar tipo_documental, solo para guardar oc referencial.

1. Corrige extracción de RUC para OC
En classifier.py, antes del fallback general de RUC, agrega:
Python
if not ruc and doc_type == "orden_compra":
    patrones_ruc_oc = [
        r"SEÑOR\(ES\)\s*:\s*.+?\bR\.?U\.?C\.?\s*:\s*([0-9]{11})",
        r"SENOR\(ES\)\s*:\s*.+?\bR\.?U\.?C\.?\s*:\s*([0-9]{11})",
        r"\bR\.?U\.?C\.?\s*:\s*[\r\n\s]*([0-9]{11})",
    ]

    for patron in patrones_ruc_oc:
        m = re.search(patron, text_u, re.IGNORECASE | re.DOTALL)
        if m:
            ruc = m.group(1)
            break

3. Sobre el archivo sospechoso 007902
Sí, ese archivo es distinto: el PDF no tiene texto extraíble y parece una imagen incrustada, mientras que el 007914 sí tiene texto digital normal. El contenido visual del 007902 muestra “Orden de Compra N°:007902”, pero al no tener texto real es compatible con un PDF rasterizado/escaneado/impreso a PDF. Eso no prueba edición, pero sí es motivo para marcarlo como “revisión visual”. �
04-0001 ORDEN_COMPRA OC 007902 20538549071 OMEGA_POWER_S_A_C.pdf
La OC 007914 sí aparece como texto digital extraíble con “Orden de Compra Nº:007914”, por eso es más confiable para extracción automática. �
04-0002 ORDEN_COMPRA OC 007914 SIN_RAZON_SOCIAL.pdf
Mi recomendación: si pdf_text viene vacío y solo se procesa por OCR, clasifícalo, pero agrega una observación tipo:
Plain text
Documento rasterizado: validar visualmente.
