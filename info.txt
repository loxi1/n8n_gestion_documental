Sí, esa es la causa probable: el PDF de Starsoft está representando el símbolo ° con una fuente/carácter que al extraer texto sale como *, º, espacio, o incluso desaparece.
Por eso la regla no debe depender del símbolo. Para OC usa una regex flexible que busque:
Plain text
ORDEN DE COMPRA N   ...   007902
sin importar qué haya entre N y el número.
Reemplaza _extract_oc_fields() por esta versión:
Python
def _extract_oc_fields(text_u: str, name_u: str) -> tuple[str | None, str | None]:
    patrones = [
        # Orden de Compra N°:007902 / N*:007902 / N : 007902 / N 007902
        r"\bORDEN\s+DE\s+COM(?:P|R)A\s+N\b.{0,40}?([0-9]{4,})\b",

        # Orden de Compra: 007902
        r"\bORDEN\s+DE\s+COM(?:P|R)A\b.{0,40}?([0-9]{4,})\b",

        # Orden Compra: 007902
        r"\bORDEN\s+COMPRA\b.{0,40}?([0-9]{4,})\b",

        # OC: 007902 / OC-007902
        r"\bOC[:\s-]*([0-9]{4,})\b",
    ]

    for fuente in (text_u, name_u):
        for patron in patrones:
            m = re.search(patron, fuente, re.IGNORECASE | re.DOTALL)
            if m:
                return "OC", m.group(1)

    return None, None
