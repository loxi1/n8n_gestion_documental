Regla clave
Para una OC de ese tipo, no uses una etiqueta suelta como “Orden Compra:” dentro de otro documento para clasificar.
La OC real debe detectarse por algo fuerte como:
ORDEN DE COMPRA N
o ORDEN DE COMPRA NRO
o ORDEN DE COMPRA N°
y además un número correlativo
Eso evita que una factura que solo menciona “Orden Compra:” se convierta por error en OC.

Ajuste conceptual importante
Como ya definiste bien tu negocio:
la factura principal define el mes del grupo
la OC solo aporta contexto
la fecha de la OC no manda la carpeta final si viene junto a factura
O sea, aunque a la OC le saques 15/04/2026, si viene con una factura principal de abril, igual hereda abril. Si llega sola, queda en pendiente_asociacion con su propia fecha guardada.
Función de extracción específica para OC
Te recomiendo incluso aislarla para que no mezcles demasiadas reglas genéricas:

def extract_oc_fields(text_u: str, text_original: str) -> dict:
    serie = None
    numero = None
    fecha_emision = None
    ruc = None
    razon_social = None
    importe = None

    m = re.search(r"\bORDEN DE COMPRA\s*N(?:RO)?[°º:\s-]*([0-9]{4,})\b", text_u)
    if m:
        serie = "OC"
        numero = m.group(1)

    m = re.search(r"\bFECHA\s*[:\-]?\s*([0-9]{2}/[0-9]{2}/[0-9]{4})\b", text_u)
    if m:
        fecha_emision = m.group(1)

    m = re.search(r"\bR\.?U\.?C\.?\s*[:\-]?\s*([0-9]{11})\b", text_u)
    if m:
        ruc = m.group(1)

    m = re.search(r"\bSEÑOR\(ES\)\s*[:\-]?\s*(.+)", text_original, re.IGNORECASE)
    if m:
        razon_social = m.group(1).strip()

    m = re.search(r"\bTOTAL\s*S/?\s*([0-9][0-9,\.]*)\b", text_u)
    if m:
        importe = m.group(1).strip()

    return {
        "tipo_documental": "orden_compra",
        "serie": serie,
        "numero": numero,
        "ruc": ruc,
        "fecha_emision": fecha_emision,
        "importe": importe,
        "razon_social": razon_social,
    }
